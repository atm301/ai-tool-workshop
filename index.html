MODULE 02 · AI 助手基礎

提示詞不是一句話：七元素框架

把需求拆成七個層級，每一項都明確 — 你能逐項檢查、逐項調整，做出來的助手才穩定、可複用。點每一列展開說明與範例。

01角色 ROLE−

設定 AI 的身份與專業背景。明確的角色，能讓回應的專業度與語氣立刻定錨。

你是一位擅長台灣市場的數位行銷顧問。
02目標 GOAL+

明確告知 AI 這個工具的核心任務，讓每一次輸出都對準同一個目的。

專門協助用戶撰寫社群貼文與廣告文案。
03背景 CONTEXT+

補充使用情境與受眾資訊，AI 才知道「對誰說、在什麼場合說」。

用戶為中小企業主，品牌以 B2C 零售為主。
04限制 CONSTRAINT+

告訴 AI 不能做什麼、要注意什麼。限制和指示一樣重要。

不使用英文 slogan，語調親切、不過度正式。
05格式 FORMAT+

指定輸出的結構與長度，結果才能穩定、可以直接拿來使用。

每次產出 3 則文案，每則 80–120 字，含 emoji。
06範例 EXAMPLE+

用 Few-shot 示範輸入與輸出，能快速校準 AI 的風格與品質。

［輸入範例貼文］→［輸出風格參考］
07反應規則 REACTION+

設定特殊情況的應對邏輯，讓工具在資訊不足時也不會亂答。

若用戶沒提供產品資訊，先主動詢問三個問題。

七元素完整示範｜社群文案 AI 助手

把七個元素串起來，就是一份完整、可直接貼到 ChatGPT / Claude / Gemini 的助手設定。

🪄 社群文案 AI 助手（完整版）複製即用 
【角色＋目標】你是一位擅長台灣市場的數位行銷顧問，專門協助我撰寫社群貼文與廣告文案。
【背景】我是中小企業主，品牌定位 B2C 零售，主要客群為 25–40 歲台灣消費者。
【限制＋格式】請避免英文 slogan，語調親切不過度正式；每次產出 3 則文案，每則 80–120 字，含適量 emoji。
【範例風格】請參考這類口吻：「最近天氣轉涼，是時候換上你的秋日新寵了 🍂」——溫度感、生活化、結尾有行動暗示。
【反應規則】若我未提供產品資訊，請先主動詢問：① 產品/服務是什麼 ② 目標受眾 ③ 想傳達的核心訊息，再開始撰寫。

提示詞範本庫

複製就能用的 提示詞範本

這是整堂課的重點外帶包。每個範本按「📋 複製」即可貼到 ChatGPT / Claude / Gemini，把 ［方括號］ 換成你的內容。

① 結合時事寫文（SOP：查時事 → 結合寫文 → 篩選）

先到 Google Trends 台灣 找最新熱門關鍵字，再套這個公式。

📰 時事行銷貼文 我是誰 / 想做什麼 / 目的
我販售［產品］，特色是［產品特色］。
我要針對近期 Google 上最熱門的時事「［時事關鍵字］」撰寫社群貼文，
目的是［銷售 / 引流 / 互動］，希望達成［具體目標，例如月銷售額 50 萬→100 萬］。
請依：1.引題痛點 2.真實情境 3.產品優勢與解方 4.限定優惠/緊迫感 5.行動呼籲 CTA 的結構撰寫，
並在結尾設計一句引導留言的互動問句。

② 一次生 10 個高點閱標題

🏷️ 標題發想
我賣［產品］，特色是［特色］，想要［目標客群］來買。
給我 10 個高點閱率的標題，風格多元（情境型、數字型、提問型、痛點型各幾個），避免農場式驚嘆號。

進階接續用：「根據第 X 個標題再產出 10 個」→「根據第 X 個標題寫社群文案」→「根據上面寫一檔行銷活動」→「根據上面設計一張社群用圖」。

③ 風格切換（同一篇文，換不同口吻）

🎭 風格變體 瓊瑤 / 金庸 / 偶像劇 / 哈利波特
用［瓊瑤體 / 金庸武俠風 / 偶像劇風 / 哈利波特魔法信風］的口吻，
重寫上面這篇貼文。保留原本的優惠資訊與行動呼籲，但換上該風格的敘事腔調與比喻。
🌊 譬喻法寫文
幫我用譬喻的方式寫一段文案：把［主題，例如旅行 / 我的產品］
比喻成［一部電影 / 一條道路 / 一場長途駕駛 / 一本還沒寫完的小說］，
寫成 3 個短版本讓我挑選，每版 4–6 行，語氣有畫面感。

④ 用 Claude 產出 PRD（開發工具前先做）

📄 PRD 產品需求文件 給 Claude
我想開發一個［工具名稱］，幫助［目標使用者］解決［他們的問題］。
請幫我產出完整的 PRD 文件，包含這 5 個欄位：
1. 工具名稱 × 核心目標（一句話）
2. 目標使用者與痛點
3. 輸入（使用者給什麼、格式）
4. 輸出（AI 回傳什麼、長什麼樣）
5. 限制（不能做什麼、邊界在哪）
若有沒想清楚的地方，請主動反問我，再把欄位補齊。
小提醒：把這些範本存進你的 GEM / GPT 的 System Prompt，就不必每次重貼。做好的助手還能分享給同事、給團隊，變成可複製的工作流程。

開發方法論

先想清楚，再動手：PRD → SSD → 開發

在寫任何 Prompt 或程式之前，先做兩份文件，能在動手前就避開大半的坑。

📄PRD 決定「要做什麼」

產品需求文件，五個必填欄位：

  • 工具名稱 × 核心目標
  • 目標使用者與痛點
  • 輸入（資訊與格式）
  • 輸出（內容與樣式）
  • 限制（邊界在哪）

📐SSD 決定「怎麼做」

系統設計文件，把需求轉成技術架構：

  • 選擇平台與模型
  • Prompt 架構（System / User / Assistant）
  • 資料流向：輸入 → 處理 → 輸出
  • 是否需要 API？是否要儲存資料？
  • 錯誤處理 × 邊界條件

🪶取得 Claude API Key

在 Anthropic Console 生成金鑰。

console.anthropic.com
  • 用 .env 儲存金鑰：寫進環境變數，絕不寫死在程式碼裡，也不上傳 GitHub。
  • 設定使用量上限：先設好上限，避免一個 bug 讓帳單瞬間爆炸。
  • 先用 Free tier 試：免費額度試水溫，確認需求後再升級付費。

資安四大核心原則（不能踩）

🔑01 金鑰保護

API Key 等於你的密碼，一律用環境變數儲存、不上傳 GitHub。

📉02 資料最小化

只輸入完成任務所需的最少資訊，別餵 AI 吃個資、機密合約。

✅03 輸出驗證

AI 說的不一定對。數字、法律、重要決策一定要人工確認。

🏢04 企業 vs 個人

免費版可能用對話訓練模型；處理公司資料請用有隱私保障的企業版。

⚠ Prompt Injection（提示詞注入）：有人在給 AI 的輸入內容裡偷藏指令，誘導 AI 忽略你的設定。四個防範：
  • 未經檢查的外部文字，不要讓 AI 直接當指令執行
  • 牽涉金錢、刪改、對外發送的動作，一律先人工確認
  • 只給工具完成任務所需的最小權限
  • AI 的產出先審核，確認無誤再發布或寄出

常見問題 Q&A

Q我不會寫程式，這些工具真的適合我嗎？
適合。今天的重點是用 PRD 思維與 AI 輔助（GEMs / GPTs / Claude），讓非工程背景的人也能完成「從想法到上線」。介面操作就能做出第一個助手。
QGemini 和 Claude，哪一個比較好用？
看任務。日常辦公、Google 生態整合用 Gemini / GEMs；寫 PRD、長文規劃、生成程式碼用 Claude。兩個搭配用最順。
QMCP 要怎麼開始安裝、設定？
最快路徑：先到 hub.twinkleai.tw 一鍵安裝、在客戶端貼上 API key；想更進階再看 modelcontextprotocol.io 的官方文件。
Q免費版和付費版差異在哪？
付費版通常解鎖更強模型、更高額度與進階功能（如 Artifacts、Deep Research）。資料隱私上，企業版會明確承諾不將資料用於訓練 — 處理公司資料請用企業版。
QAPI 費用大概怎麼估算？
依「呼叫量 × 模型單價」計費，先用 Free tier 試水溫，並在後台設使用量上限。小型工具個人試玩通常落在免費額度內。
